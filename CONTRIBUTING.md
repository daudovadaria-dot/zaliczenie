# Contributing

## Zasady współpracy w projekcie Zoo Online

Ten projekt jest przeznaczony do nauki pracy z systemem kontroli wersji Git oraz GitHub.

---

## Nazewnictwo branchy

Każda nowa gałąź powinna być tworzona według schematu:

Przykłady:
- lew-jan
- tygrys-ania
- pantera-daria

---

## Zasady commitów

Commity powinny być krótkie i opisowe.

Przykłady:
- Dodano sekcję lwa
- Dodano zdjęcie pantery
- Poprawiono opis tygrysa

---

## Pull Request

Każda zmiana musi być zgłaszana przez Pull Request do gałęzi main.

W opisie PR należy krótko opisać wprowadzone zmiany.

---

## Komunikacja

Wszelkie problemy i pytania powinny być zgłaszane przez GitHub Issues lub bezpośrednią komunikację w zespole.

---

## Zasady pracy

- nie pracujemy bezpośrednio na main
- każda funkcja = osobna gałąź
- każdy branch musi mieć Pull Request
